# STRICT STRUCTURE RULES

## RULE 1:
-Parent nodes are organizational containers only.
-If content exists entirely inside children:
-Parent text MUST be empty.
-Correct:
{

"text":"",
"children":[
{

...
}
]
}
-Wrong:
{
"id":"Q5",
"text":"(a) Name process ... (b)...",
...
}

## RULE 2:
-If a question contains (a),(b),(c), OR (i),(ii),(iii) or any other numbering style, create child nodes.
-Do NOT duplicate child text into parent.
-Wrong:
Q1.text = full question text
Q1.a.text = repeated text
Correct:
Q1.text=""
Q1.a.text="..."
Q1.b.text="..."

## RULE 3:
-Parent text exists ONLY when there is actual shared text before children.

### Correct Example:
Q14.text="Explain the following in brief:"
Q14.children:
Q14.a.text="What is JSON?"
Q14.b.text="What is Javascript?"

-Reason:
Scenario paragraph is shared. In this example both children questions must be answered in brief.

### Wrong example:
Q14.text=""
Q14.children:
Q14.a.text="Explain the following in brief: What is JSON?"
Q14.b.text="Explain the following in brief: What is Javascript?"
this example is wrong.

## RULE 4:
-Attachments belong to the lowest relevant node.
-Wrong:
Q5.attachments=[diagram]
-Correct:
Q5.a.attachments=[diagram]
because diagram belongs only to part a.

## RULE 5:
-Never move attachments upward.
-Never dilute attachments to parent.
